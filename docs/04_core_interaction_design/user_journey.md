## 一、用户画像（Persona）

### Persona A：「跳槽焦虑者」— 小陈
- **背景**：后端开发，工作3年，想从传统企业跳到互联网大厂
- **核心痛点**：不知道自己简历差在哪、面试经常在技术轮被刷、不知道该补哪些知识
- **使用动机**：投了20份没回音，想搞清楚问题出在哪
- **行为特征**：目的导向强，希望得到具体可执行的反馈，不喜欢泛泛而谈

### Persona B：「应届竞争者」— 小林
- **背景**：计算机专业应届生，项目经验少，简历薄
- **核心痛点**：不知道如何把学校项目写得有竞争力、没有面试经验很紧张
- **使用动机**：秋招季，想在面试前充分练习
- **行为特征**：时间充裕，愿意反复练习，对鼓励性反馈敏感

## 二、功能1：简历优化旅程

### 2.1 旅程概览

```
入口 → 上传简历 → 绑定JD → [系统后台分析] → 查看评分报告 → 查看改写建议 → 决策 → 导出/继续
```

### 2.2 完整旅程步骤

| 步骤 | 用户行为 | 系统响应 | 用户情绪 | 关键指标 |
|------|---------|---------|---------|---------|
| **0. 入口** | 从首页/面试报告点击「优化简历」| 跳转简历库页 | 期待 | — |
| **1. 上传简历** | 拖拽或点击上传 PDF/DOCX | 立即显示文件名 + 进度条；后台触发文本提取任务 | 平静 | 上传成功率 |
| **2. 绑定 JD** | 粘贴目标 JD 文本 / 跳过（通用分析）| 实时字数统计；超过5000字时提示截断 | 平静 | JD 填写率 |
| **3. 等待分析** | 停留在等待页，可查看进度 | Loading 动画 + 分步骤进度提示（「正在提取关键词…」「正在计算匹配度…」）| 略焦虑→有信心 | 分析完成时长（目标<30s）|
| **4. 查看综合评分** | 滚动浏览评分卡片 | 圆形评分环（0-100）+ 4个维度条形图（相关性/关键词/结构/量化）| 惊讶 / 认同 | 评分页停留时长 |
| **5. 查看差距详情** | 点击各维度展开 | 命中关键词（绿色）vs 缺失关键词（红色）高亮展示 | 恍然大悟 | 关键词点击率 |
| **6. 阅读改进建议** | 查看 Top3 优先改进建议 | 文字建议 + 每条对应「为什么重要」 | 有方向感 | 建议点击率 |
| **7. 查看改写版本** | 点击「一键改写」| SSE 流式输出改写内容，逐字出现（约30-60s）| 兴奋→期待 | 改写触发率 |
| **8. 对比原文** | 切换「原文 / 改写」对比视图 | DiffViewer 高亮显示变化部分（新增绿色、删除红色）| 评估中 | 对比视图使用率 |
| **9. 决策** | A. 点击「接受改写」B. 手动编辑 C. 拒绝 | A. 保存改写版本；B. 进入编辑模式；C. 保留原文 | 掌控感 | 改写接受率（核心指标）|
| **10. 导出/继续** | 下载简历 / 直接用于创建面试 | 提供 PDF 导出；或一键跳转「基于此简历开始面试」| 完成感 | 导出率、跳转面试率 |

### 2.3 支线：多次优化

```
用户换了新 JD → 简历库选择已有简历 → 输入新 JD → 重新评分（不重新上传）→ 对比两次评分差异
```

### 2.4 情绪曲线

```
情绪高 ──────────────────────────────────────────────────────
         改写流式输出（兴奋）   接受改写（满足）
         
情绪中 ──────  绑定JD  ──  查看评分  ──  阅读建议  ──────────
                                  
情绪低 ── 上传等待（焦虑）    看到低分（沮丧）
──────────────────────────────────────────────────────────
         步骤1       步骤3      步骤4      步骤6     步骤9
```

### 2.5 关键决策点

| 决策点 | 用户选项 | 设计原则 |
|--------|---------|---------|
| 是否填写JD | 填写 / 跳过通用分析 | 跳过时说明「填写JD可获得针对性评分」，降低门槛 |
| 接受改写与否 | 接受 / 编辑 / 拒绝 | 三选项平级展示，不用「接受」引导 |
| 改写后导出还是去面试 | 导出 / 立即面试 | 两个 CTA 并列，面试按钮主色 |

## 三、功能2：模拟面试旅程

### 3.1 旅程概览

```
入口 → 配置会话(模式+难度) → [系统初始化] → 开始面试 → 回答N题(循环) → 结束面试 → [系统生成报告] → 查看报告 → 生成训练计划(可选)
```

### 3.2 完整旅程步骤

| 步骤 | 用户行为 | 系统响应 | 用户情绪 | 关键指标 |
|------|---------|---------|---------|---------|
| **0. 入口** | 导航栏「开始面试」/ 简历优化后跳转 | 面试配置页 | 期待/紧张 | — |
| **1. 选择面试模式** | 选择：简历+JD / 仅JD / 仅简历 / 自由主题 | 对应模式的说明文字展示 | 思考 | 各模式使用占比 |
| **2. 配置难度** | 选择：简单/中等/困难/自适应 | 说明各难度的题型分布比例 | 自我评估 | 难度分布 |
| **3. 配置题数** | 选择3/5/8/10题或自定义 | 预估时长显示（每题约2-3分钟）| 权衡 | 平均题数 |
| **4. 选择模式** | 文字模式 / 语音模式 | 语音模式额外提示「需允许麦克风权限」| 习惯选择 | 语音/文字比例 |
| **5. 确认开始** | 点击「开始面试」| 全屏面试界面；顶部进度条；第一题出现（打字机效果）| 正式感/紧张 | 配置→开始转化率 |
| **6. 阅读题目** | 阅读题目，查看建议作答时长 | 倒计时（软性，超时不强制提交）；题型标签 | 思考 | 答题前等待时长 |
| **7A. 文字回答** | 在输入框输入答案 | 字数统计；语法高亮（可选）| 专注 | 平均回答字数 |
| **7B. 语音回答** | 按住录音按钮说话 | 波形动画；实时音量反馈；松手触发转写 | 紧张→自然 | 语音完成率 |
| **8. 提交回答** | 点击提交/松开录音 | SSE 流式评分开始：先显示「评分中…」→ 逐字输出评分反馈 | 期待→审视 | 提交率（完成率）|
| **9. 查看单题评分** | 阅读评分（4维度 + 亮点 + 建议）| 单题评分卡；参考答案要点（折叠，需点击展开）| 反思 | 参考答案展开率 |
| **10. 下一题** | 点击「下一题」| 新题出现；进度更新；自适应模式静默调整难度 | 重新聚焦 | 继续率 |
| **11. 结束面试** | 完成所有题 / 主动结束 | 弹框确认；触发报告生成任务（后台）| 释然/期待 | 完整完成率 |
| **12. 等待报告** | 等待页（10-30s）| 进度动画 + 「正在汇总各维度表现…」 | 期待 | 报告生成时长 |
| **13. 查看综合报告** | 浏览报告 | 雷达图（4维度）+ 整体评价 + 题目回顾手风琴 | 客观认知 | 报告完读率 |
| **14. 题目回顾** | 展开各题的完整评分 | 题目 + 我的回答 + 评分 + 参考要点 | 复盘 | 回顾点击率 |
| **15. 生成训练计划** | 点击「基于本次面试生成学习计划」| 跳转训练模块，后台生成计划 | 行动力 | 报告→训练转化率（核心指标）|

### 3.3 自适应难度体验

用户不感知档位切换的具体逻辑，只看到：
- 题目整体感觉「越来越有挑战」或「稍微简单了一些」
- 最终报告中显示「本次面试实际难度分布：简单20%/中等60%/困难20%」

### 3.4 语音模式特殊路径

```
允许麦克风 → 录音 → 转写文本出现 → 用户可编辑转写结果 → 确认提交 → 正常评分流程
              ↓ 拒绝麦克风权限
              → 提示引导开启权限 / 降级为文字模式
```

### 3.5 情绪曲线

```
情绪高 ──────────────────────────────────────────────────── 
             开始（兴奋）    好题得高分（自信）    报告雷达图（客观）
             
情绪中 ──  配置阶段  ──────────────────────  查看报告  ──────
                                  
情绪低 ──────────────── 回答困难题（压力）  看到低分（挫败→反思）
──────────────────────────────────────────────────────────
         步骤1-4    步骤6      步骤7-9         步骤13
```

## 四、功能3：知识缺口训练旅程

### 4.1 旅程概览

```
入口(面试报告/手动) → [后台生成计划] → 查看全局蓝图 → 选择知识点 → Layer1-4 学习 → 费曼总结 → (通过→下一个 / 失败→追问循环) → 完成全部 → 复测 → 查看进步
```

### 4.2 完整旅程步骤

| 步骤 | 用户行为 | 系统响应 | 用户情绪 | 关键指标 |
|------|---------|---------|---------|---------|
| **0. 入口** | A. 面试报告页点击「生成学习计划」B. 导航直接进入 | A. 后台已触发生成，直接跳转；B. 选择来源（面试报告/手动输入JD）| 主动 | — |
| **1. 等待蓝图生成** | 等待页（15-30s）| 动画 + 「正在分析你的知识缺口…构建关联图…」 | 期待 | 生成时长 |
| **2. 查看全局蓝图** | 浏览知识点关联图 | 有向图可视化：节点=知识点（颜色深浅=严重程度），边=依赖/关联关系；右侧显示学习路径列表 + 总时长估算 | 整体认知 | 蓝图停留时长 |
| **3. 理解关联关系** | 鼠标悬停/点击节点 | Tooltip 显示：why_important + severity + prerequisite_topics | 理解 | 节点交互率 |
| **4. 选择起点** | 系统推荐第一个节点（已按拓扑排序）/ 用户手动选择 | 节点进入动画；右侧面板展开学习内容 | 进入状态 | 推荐路径采纳率 |
| **5. Layer1 - 是什么** | 阅读核心概念 | SSE 流式输出概念内容（约200字）；末尾出现 connection_prompt 提示语 | 建立概念 | Layer1 完读率 |
| **6. Layer2 - 为什么** | 继续阅读 | 原理与机制；末尾引导联想与已学知识的关系 | 深化理解 | Layer2 完读率 |
| **7. Layer3 - 怎么用** | 阅读代码示例 | 带语法高亮的代码块；可展开查看完整示例 | 实践理解 | 代码块展开率 |
| **8. Layer4 - 关联什么** | 阅读关联内容 | 「你在X中学过的Y，在这里体现为…」；点击可跳转到关联知识点 | 融会贯通 | 关联点击率 |
| **9. 自测题** | 尝试回答练习题 | 题目展示（不强制，可展开参考答案）| 自我检验 | 自测完成率 |
| **10. 提交费曼总结** | 在输入框用自己的语言写总结（100-500字）| 字数提示；「用自己的话，不要复制原文」引导语 | 整理思路 | 费曼提交率（关键指标）|
| **11. 等待评估** | 等待（5-10s）| 「正在评估你的理解程度…」| 期待 | — |
| **12A. 通过费曼检验** | 查看正向反馈 | 评分详情（清晰度/准确性/完整性）+ 鼓励语 + 「解锁下一个知识点」| 成就感 | 首次通过率 |
| **12B. 未通过检验** | 查看追问提示 | 「你提到了X，但是关于Y还能深入吗？」（不给答案，给提示）| 受挫→思考 | 二次尝试率 |
| **13. 多次失败处理** | 第3次仍未通过 | 「建议你重读Layer2，特别是…部分」+ 允许跳过按钮 | 理性接受 | 跳过率 |
| **14. 完成单个知识点** | 所有层学完 + 费曼通过 | 蓝图中该节点变为绿色；显示「已掌握」；推荐下一个节点 | 递进成就感 | 单点完成率 |
| **15. 完成全部计划** | 所有节点完成 | 庆祝动画；进入复测入口 | 成就感 | 计划完成率（核心指标）|
| **16. 复测** | 逐题回答复测题（比学习时更简短）| 题目展示 → SSE 评分 → 显示与学习前的对比分 | 验证 | 复测参与率 |
| **17. 查看进步曲线** | 浏览进步报告 | 各知识点「学习前/学习后」对比图；整体提升分数 | 成就感/信心 | 分享率（社会化）|

### 4.3 费曼循环细节

```
提交总结
    ↓
系统评估
    ↓ passed=true                    ↓ passed=false（第1-2次）
解锁下一节点                         显示 follow_up_prompt
    ↓                                    ↓
继续学习                         用户补充/修改总结
                                         ↓
                                    重新提交（最多3次）
                                         ↓ 第3次仍未通过
                                    建议重读 + 允许跳过
```

### 4.4 知识图谱导航规则

| 规则 | 说明 |
|------|------|
| 前置未完成 | 有未完成前置节点时，目标节点显示「锁定」，鼠标悬停提示需先学哪些 |
| 推荐顺序 | 系统按拓扑排序高亮推荐下一个节点，但用户可自由选择解锁的节点 |
| 已完成节点 | 绿色填充，可回顾但不再强制费曼总结 |
| 进行中节点 | 蓝色边框，表示已开始未完成 |

### 4.5 情绪曲线

```
情绪高 ──────────────────────────────────────────────────────
                费曼通过（成就感）    完成全部（成就感高峰）
             
情绪中 ──  看蓝图（认知）── Layer学习（稳定）──────────────── 
                                  
情绪低 ──────────────────────────── 费曼失败（受挫）─────────
──────────────────────────────────────────────────────────
         步骤2       步骤5-8         步骤12B      步骤15
```

## 五、跨功能跳转链路

跨功能跳转是平台「闭环」体验的核心，以下为关键跳转路径：

### 5.1 链路闭环图

```
                    ┌─────────────────┐
                    │    简历优化      │
                    │ /resumes/:id/   │
                    │  optimize       │
                    └────────┬────────┘
                             │ 「基于此简历开始面试」
                             ▼
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│  简历库          │   │    模拟面试       │   │  知识缺口训练    │
│ /resumes        │   │ /interviews/new  │   │ /training/:id   │
└─────────────────┘   └────────┬────────┘   └─────────────────┘
         ▲                     │ 面试完成后                    ▲
         │                     │ 「生成学习计划」               │
         │                     └────────────────────────────▶ │
         │                                                     │
         └──────────────────────────────────────────────────── ┘
              复测完成后「优化简历」（将学习成果体现在简历中）
```

### 5.2 跳转触发点清单

| 来源页面 | 触发动作 | 目标页面 | 携带数据 |
|---------|---------|---------|---------|
| 简历优化页 | 点击「基于此简历开始面试」| 面试配置页 | `resume_id` 预填 |
| 面试报告页 | 点击「生成学习计划」| 训练计划页（加载中）| `session_id` → 后台触发 |
| 面试报告页 | 点击「重新优化简历」| 简历优化页 | `resume_id` + 面试中暴露的弱点作为 `rewrite_hints` |
| 训练计划完成 | 点击「验证学习成果-去面试」| 面试配置页 | 建议 `jd_only` 模式 |
| 训练复测完成 | 点击「将进步反映到简历」| 简历优化页 | 携带 `training_plan_id`，改写时参考新掌握技能 |
| 仪表板 | 点击各功能入口卡片 | 对应功能页 | — |

### 5.3 新用户引导流程（Onboarding）

```
注册/登录
    ↓
引导弹窗（3步）：
  Step1: 「上传你的简历，我来帮你找出问题」→ 上传
  Step2: 「粘贴你的目标JD，获取针对性建议」→ 粘贴或跳过
  Step3: 「选择：先优化简历 / 先模拟面试」→ 分叉
    ↓                        ↓
简历优化主流程          面试配置页（引导选模式）
```

## 六、通用交互规范

### 6.1 加载状态规范

| 等待时长 | 交互模式 |
|---------|---------|
| < 1s | 无 Loading，直接呈现结果 |
| 1-3s | Skeleton 骨架屏 |
| 3-30s | 进度动画 + 步骤提示文案（「正在…」）|
| > 30s | 进度动画 + 「可以先做其他事，完成后通知你」+ 后台通知机制 |

### 6.2 SSE 流式展示规范

| 场景 | 展示行为 |
|------|---------|
| 简历改写 | 逐字打字机效果，光标闪烁 |
| 面试评分 | 先出现维度分数，再逐段输出文字反馈 |
| 学习内容 | 按 Layer 分块输出，每块完整后再出下一块 |
| 费曼评估 | 先出现分数，再流式输出反馈文字 |

### 6.3 错误提示规范

| 错误类型 | 展示方式 | 文案风格 |
|---------|---------|---------|
| 网络中断 | Toast 红色提示 + 自动重试 | 「网络不稳定，正在重连…」|
| LLM 超时 | 页面内 Banner + 重试按钮 | 「AI 分析超时，点击重试」|
| 文件格式错误 | 上传区域内联提示 | 「仅支持 PDF 和 DOCX 格式」|
| SSE 中断 | 已输出内容保留，出现重连按钮 | 「内容生成中断，点击继续」|
| 权限不足 | Modal 弹窗 | 「请先登录 / 该功能需要上传简历」|
