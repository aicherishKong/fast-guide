# 04 核心交互链路设计

> 阶段目标：设计用户在平台内的核心操作路径，确保功能闭环。

## 输出文件清单

| 文件 | 内容 | 状态 |
|------|------|------|
| `user_journey.md` | 三大功能的用户旅程地图 | 待完成 |
| `interaction_flow.md` | 关键操作的交互流程图（含异常路径）| 待完成 |
| `state_transitions.md` | 核心实体状态机（简历、面试会话、训练计划）| 待完成 |

## 三大核心链路（草稿）

### 链路1：简历优化
```
上传简历 → 输入/选择JD → 查看评分报告 → 查看改写建议 → 接受/拒绝改写 → 导出简历
```

### 链路2：模拟面试
```
选择配置(模式+难度) → 进入面试 → 回答问题(文字/语音) → 查看实时评分 → 结束面试 → 查看报告 → 生成训练计划
```

### 链路3：知识缺口训练
```
查看全局蓝图 → 选择知识点 → 学习4层内容 → 提交费曼总结 → 通过检验 → 复测 → 查看进步曲线
```

## 完成标准

- [ ] 三大功能链路的正常路径 + 异常路径都已覆盖
- [ ] 关键状态转移图完成（简历状态、会话状态、训练项状态）
- [ ] 跨功能跳转路径明确（面试完成 → 自动生成训练计划）
