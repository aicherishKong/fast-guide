## 二、布局架构

### 2.1 全局布局结构

```
┌─────────────────────────────────────────────┐
│  Header（顶部导航栏）                         │
│  - Logo / 平台名                             │
│  - 主导航链接（工作台 / 简历 / 面试 / 训练）   │
│  - 用户头像 + 下拉菜单（个人信息 / 退出）      │
├──────────┬──────────────────────────────────┤
│ Sidebar  │  主内容区（Main Content）          │
│（部分页面）│  - 页面级组件渲染区               │
│          │  - 滚动容器                       │
└──────────┴──────────────────────────────────┘
```

**布局规则：**
- 认证页（P-01, P-02）：无 Header、无 Sidebar，居中卡片布局
- 面试会话页（P-09）：全屏覆盖，隐藏 Header 和 Sidebar，沉浸式模式
- 其余页面：统一使用 `Layout` 组件（Header + 可选 Sidebar + Main）

## 四、核心交互规范

### 4.1 流式内容渲染（SSE）

**触发场景：** 简历改写、面试评分输出、训练内容加载、费曼评估反馈

**交互规则：**
- 触发 SSE 请求后，按钮立即变为 Loading 状态，禁止重复触发
- 内容逐字出现（StreamingText 组件），光标闪烁效果
- 收到 `{"done": true}` 后：光标消失，操作按钮恢复可用
- 收到 `{"error": "..."}` 后：显示错误提示，提供「重试」按钮
- SSE 流过程中禁止页面跳转（路由守卫拦截），提示用户「正在生成中，确定离开？」

**加载状态文案规范：**

| 场景 | 加载文案 |
|------|---------|
| 简历分析中 | 「正在提取关键词…」→「正在计算匹配度…」→「分析完成」|
| 简历改写中 | 「正在为你改写简历…」|
| 面试评分中 | 「正在评估你的回答…」|
| 训练内容生成中 | 「正在生成学习内容…」|
| 费曼评估中 | 「正在分析你的理解…」|

### 4.2 arq 后台任务通知

**触发场景：** 简历分析完成、面试报告生成完成、训练计划生成完成

**交互规则：**
- 任务提交后，当前页展示「进行中」状态卡片（轮询或 WebSocket 通知）
- 任务完成后，通过 WebSocket `/ws/notifications` 接收推送
- 展示 Toast 通知（右下角，自动消失5s）：「简历分析完成，点击查看」
- Toast 可点击跳转至对应结果页

### 4.3 文件上传

**约束：**
- 支持格式：PDF / DOCX
- 大小限制：≤ 10MB
- 前端校验（格式 + 大小）在上传前执行，不依赖后端返回错误

**交互流程：**
- 拖拽进入区域：边框高亮蓝色，显示「松手即可上传」
- 选择文件后：立即显示文件名 + 大小，显示「上传并分析」按钮
- 上传中：线性进度条，显示上传百分比
- 上传完成：进度条满格，提示「上传成功，正在分析…」，跳转等待页

### 4.4 语音录制

**权限处理：**
- 点击「语音模式」时主动申请麦克风权限
- 用户拒绝权限：展示提示，引导用户在浏览器设置中开启，提供「切换文字模式」选项

**录音交互：**
- 「开始说话」按钮：点击开始录音，按钮变红，麦克风图标动效
- 实时波形动画：展示正在输入的音频
- VAD（语音活动检测）：静音超过2秒，自动停止录音并提交
- 「停止」按钮：手动停止，触发提交
- 转写结果：实时在录音区下方预览转写文本

### 4.5 知识图谱交互（P-12）

**节点操作：**
- 悬停节点：显示 Tooltip（主题名 / 预估学习时长 / 当前状态）
- 点击节点：跳转至对应 ItemPage
- 节点颜色：灰色=待学习，蓝色=进行中，绿色=已完成，深绿=已掌握（复测通过）
- 边类型：实线=前置依赖（必须先学），虚线=横向关联（可参考），点线=延伸方向（可选）

**图谱操作：**
- 支持缩放（滚轮 / 触控板双指）
- 支持拖拽移动（鼠标拖拽 / 触控板滑动）
- 「居中」快捷按钮：重置视图位置

### 4.6 面试自适应难度提示

**规则：** 连续2题均分 > 80 升档；连续2题均分 < 55 降档
**交互：**
- 档位变化时，顶部进度条旁出现临时提示：「难度已提升至：中等」（3秒后消失）
- 当前难度标签随时可见（右上角角标）
- 不打断答题流程，无模态弹窗

### 4.7 费曼学习循环

**强制规则：** 学习完4层内容后，费曼总结输入框必须填写并提交，方可进入下一知识点。

**交互：**
- 字数不足100字时，「提交总结」按钮禁用，显示「至少需要100字，当前X字」
- passed=false（最多3次）：显示追问提示，输入框清空，重新填写
- 第3次仍未通过：显示「允许跳过」按钮（文字链接样式，非主色，降低突显度）
- passed=true：显示庆祝动效（小彩纸），「下一个知识点」按钮变主色

### 4.8 空状态与错误处理

| 场景 | 空状态文案 | 操作引导 |
|------|----------|---------|
| 简历库无简历 | 「还没有简历，上传第一份吧」| 「上传简历」按钮 |
| 面试历史为空 | 「还没有面试记录，开始第一次面试」| 「开始面试」按钮 |
| 训练计划为空 | 「完成一次面试后，即可生成专属训练计划」| 「去面试」按钮 |
| 网络错误 | 「网络连接异常，请检查后重试」| 「重试」按钮 |
| 服务器错误（5xx）| 「服务暂时不可用，请稍后再试」| 「重试」按钮 |
